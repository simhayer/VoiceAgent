## 2024-05-24 - N+1 query vulnerability with CallLogs and CallMessages
**Learning:** The `/call-logs` endpoint returned the call log objects and used a loop to iteratively perform a separate query to fetch `CallMessage` records for each log. The `CallLog.messages` relationship specifies `order_by="CallMessage.sequence"`. When eagerly loading `CallLog.messages` using `selectinload`, it automatically maintains the correct sequence sorting, avoiding the N+1 query.
**Action:** Always scan loops containing database calls. In SQLAlchemy, use `.options(selectinload(...))` for loading to-many relationships rather than explicit query blocks inside loops, but ensure `from sqlalchemy.orm import selectinload` is properly imported in the scope to avoid `NameError` exceptions at runtime.
